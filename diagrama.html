🧠 AI Proactive Agent

Arquitectura y flujo de datos de la extensión

📁 Estructura del proyecto

manifest.json
JSON Configuración de la extensión (Manifest V3)
background.js
JS Service worker: captura pantalla + llama a API IA
content.js
JS Inyecta el widget de chat en cada página
styles.css
CSS Estilos del chat flotante
popup.html
HTML Popup de control rápido
popup.js
JS Lógica del popup
options.html
HTML Página de configuración
options.js
JS Lógica de configuración
icons/
PNG Íconos 16x16, 48x48, 128x128

🔗 Interacción entre componentes

🌐 Página web donde navega el usuario
← inyecta →
💬 content.js Widget chat flotante
← chrome.runtime →
⚙️ background.js Service worker
← fetch →
☁️ API IA GPT / Gemini / DeepSeek
content.js se comunica con background.js via chrome.runtime.sendMessage
background.js captura la pantalla (chrome.tabs.captureVisibleTab) y llama a la API

🔄 Flujo de datos paso a paso

1
Navegación / Click

El usuario navega a una página nueva o cliquea "Enviar" en el chat

content.js
2
Extraer texto de la página (solo si modelo no tiene visión)

content.js clona el DOM y limpia scripts/styles/nav para obtener el texto útil

(opcional)
3
Enviar mensaje al background

{ type: 'ANALYZE_SCREENSHOT', query: '...', history: '...' }

chrome.runtime.sendMessage
4
Capturar pantalla

background.js obtiene el tabId del sender y llama a chrome.tabs.captureVisibleTab

background.js
5
Construir payload de la API

Combina: System Prompt + Historial conversación + Mensaje usuario + Imagen JPEG

background.js
6
Llamar a la API de IA

Envía vía fetch() al endpoint configurado (OpenAI / Gemini / DeepSeek)

fetch → API
7
Parsear respuesta

Separa el texto de la IA de las sugerencias (separadas por ||)

background.js
8
Enviar resultado al content script

{ response: '...', suggestions: ['...', '...'] }

sendResponse
9
Renderizar en el chat

content.js muestra el mensaje y las sugerencias como chips cliqueables

content.js
10
Guardar en historial

Agrega el par (usuario → asistente) al array conversationHistory[]

content.js

📦 Formatos de mensajes

content.js → background.js

{ type: "ANALYZE_SCREENSHOT", // Primer mensaje o cuando hay nueva pregunta query: "¿Qué significa este gráfico?", // Historial de la conversación (texto plano) history: "Usuario: Hola\nAsistente: Bienvenido..." }

background.js → API de IA (OpenAI-compatible)

{ // System prompt configura por el usuario messages: [ { role: "system", content: "Eres un asistente..." }, // Historial como contexto (si existe) { role: "user", content: "Historial: ..." }, // Mensaje actual + imagen { role: "user", content: [ { type: "text", text: "¿Qué significa?" }, { type: "image_url", image_url: { url: "data:image/jpeg;base64,..." } } ]} ], model: "gpt-4o-mini", temperature: 0.7 }

API → background.js → content.js

{ // Respuesta principal de la IA response: "Este gráfico muestra las tendencias de ventas...", // Sugerencias extraídas después del último || suggestions: [ "¿Cómo se comparan estos datos con el año pasado?", "¿Qué factores explican el pico de marzo?" ] }

🏗️ Capas de la arquitectura

🎨 Capa de UI

  • content.js — Widget chat
  • styles.css — Diseño visual
  • popup.html/js — Menú rápido
  • options.html/js — Configuración

⚙️ Capa de servicio

  • background.js — Service Worker
  • chrome.tabs.captureVisibleTab
  • chrome.storage.sync
  • Manejo de mensajes

☁️ Capa de IA

  • OpenAI (GPT-4o, etc.)
  • Google Gemini
  • DeepSeek
  • Cualquier API compatible
Las capas se comunican mediante chrome.runtime.sendMessage (UI ↔ Servicio) y fetch (Servicio ↔ API)
